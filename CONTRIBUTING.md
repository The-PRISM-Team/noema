# Contributing to Project Noema

Thanks for your interest in contributing! This file contains minimal guidance to get started.

- Code style: use the repository's existing style. A pre-commit linter/formatter will auto-fix whitespace and indentation issues.
- Workflow: fork the repo (or branch from `main`), make changes on a topic branch, run the linter, and open a pull request with a clear description.
- Issues: open an issue for any bug, feature request, or proposal before starting larger work.
- Tests: none are required at the moment, but include a short manual verification note in PR descriptions when relevant.
- Communication: reference `COLLABORATION.md` for more about reviews and expectations.

Small PRs (single feature/fix) get reviewed faster — prefer focused changes.
